# 目标岗位库 Schema（target_library.csv）

> 本文件是 `boss-zhipin-copilot` 的**数据契约**：检索 / 去重 / 入库的唯一权威数据源。
> 默认位置：`./target_library.csv` 或 `$LIB_CSV` 指向的文件。
> 脚本按**表头字典键**映射，可增减列，不依赖列序号。

---

## 一、库的地位

- 任何「已收藏 / 已沟通 / 放弃」状态变更，必须回写本库（真实结果，不记假数据）。
- 跨运行去重以 `URL` 为唯一主键：已入库 URL 不再重复收藏。
- 本库与 `profile.yaml` 解耦：`profile` 决定「收不收」，`library` 只记录「收了什么」。

---

## 二、推荐表头（29 列以内，可裁剪）

| 列名 | 说明 |
|---|---|
| 岗位ID | 站内编号（如 `BJ-0001`），按库最大号 +1 起算 |
| 岗位名 | 岗位名 |
| 公司名 | 公司名 |
| 公司规模 | 如 `100-499人` / `10000人以上` |
| 公司阶段 | 成长期 / 不需要融资 / 未融资 / 已上市 / 天使轮 / … |
| 行业 | 行业标签 |
| 城市 | 工作城市 |
| 薪资 | 原始薪资文本（如 `25K` / `40-60K·14薪`） |
| 经验要求 | 如 `5-10年` |
| 类型 | 如 `策略产品` / `增长` / `C端` / `AI应用` |
| 状态 | 见下方状态词表 |
| URL | 岗位详情页 URL（去重主键） |
| 评分 | `filter_library.py` 计算的 0–100 分 |
| 排除原因 | 被硬排除/未过门槛的原因（通过则为空） |
| 招聘方 | 真实招聘方姓名（来自 `.job-boss-info .name`，**非登录账号**） |
| 更新时间 | 最近一次写入/变更时间（ISO） |

---

## 三、状态取值（子串匹配跳过）

过滤器用子串匹配以下词跳过已处理行：

- `已沟通`（含 `已沟通-已读不回` / `已沟通-已送达` / `已沟通-已回复` / `已沟通-已婉拒` 等）
- `已收藏`（含 `已收藏(未沟通)` / `已收藏(感兴趣)`）
- `放弃`（含 `已放弃` / `放弃`）

> 入库时只写真实结果：书签成功写 `已收藏(感兴趣)`，发送成功写 `已沟通-已送达`，
> 用户放弃写 `放弃`。**绝不记假数据**。

---

## 四、与 profile 的关系

- `profile.hard_exclude` / `profile.thresholds` / `profile.boost_keywords`
  由 `filter_library.py` 读取，决定每行「收 / 不收 / 评分多少」。
- 本 schema 不预置任何城市的薪资硬编码——一切由 `profile` 决定。

---

## 五、初始化

- 首次使用：`bash scripts/setup_library.sh` 按模板生成空 CSV（含表头）。
- 或复制 `assets/target_library_template.csv` 自行改名。
